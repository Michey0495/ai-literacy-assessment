## Cursorへの命令書（プロンプト）

あなたは世界最高峰のフロントエンドエンジニア兼UI/UXデザイナーです。
以下の仕様とデータに基づき、**「生成AI活用能力アセスメントアプリケーション」**を構築してください。

### 1. プロジェクト概要

ユーザーの生成AIに対するリテラシー、プロンプト技術、リスク管理能力などを測る30問のテストを行い、結果を多角的に分析してレーダーチャートとフィードバックを表示するWebアプリです。

### 2. 技術スタック

* **Framework:** Next.js (App Router)
* **Language:** TypeScript
* **Styling:** Tailwind CSS (美しい、モダンで信頼感のあるデザイン)
* **Components:** shadcn/ui (推奨) または Headless UI
* **Charts:** Recharts (レーダーチャート描画用)
* **Icons:** Lucide React

### 3. 要件定義

#### A. データ管理

* 後述する「問題データ（CSV形式）」をパースして使用する、もしくは `questions.ts` として定数管理する構造にしてください。
* 各問題には `category` が設定されており、これが分析の軸になります。

#### B. UI/UXフロー

1. **トップページ:**
   * タイトル「生成AIコンピテンシー・スキャン」
   * 説明文「全30問であなたの生成AI活用力を5つの次元で分析します」
   * 「診断を開始する」ボタン
2. **テスト画面:**
   * 1問ずつ表示（カード形式）。
   * プログレスバー（現在地/30問）を表示。
   * 選択肢をクリックすると次の問題へ遷移。
   * **裏側の仕組み:** ユーザーの回答ログを蓄積する。
3. **結果画面（分析レポート）:**
   * **総合スコア:** 100点満点。
   * **レベル判定:** スコアに応じて「Lv.1 危険水域」～「Lv.4 エキスパート」を表示。
   * **レーダーチャート:** 以下の5つの軸で得点率（%）を可視化。
     1. 原理・リテラシー (Principles)
     2. プロンプト技術 (Prompt Engineering)
     3. リスク・倫理 (Risk & Ethics)
     4. 業務適用 (Business Application)
     5. 創造性・高度活用 (Creativity & Advanced)
   * **フィードバック:** 得点が低かった項目に対する具体的なアドバイスを表示。

#### C. ロジック仕様（重要）

* 各カテゴリは6問ずつ設定されています。
* カテゴリごとのスコア = (正解数 / 6) * 100
* チャートにはこのカテゴリごとのスコアをマッピングしてください。
